简介

例如一个日志系统，很容易使用RabbitMQ简化工作量，一个Consumer可以进行消息的正常处理，另一个Consumer负责对消息进行日志记录，只要在程序中指定两个Consumer所监听的queue以相同的方式绑定到同一个exchange即可，剩下的消息分发工作由RabbitMQ完成。

使用RabbitMQ server

  • ErLang语言包；
  • RabbitMQ安装包；
  • RabbitMQ同时提供了java的客户端（一个jar包）；
  • RabbitMQ WebManager,网页管理界面。

基本概念

  • exchange 交换机 消息直接发送到交换机，队列绑定交换机的方式主要有两种，一个是direct，按照routekey绑定到队列，一种是topic，按照规则转发消息。一个交换机可以绑定多个队列，一个队列可以被多个交换机绑定。topic会指定一个binding key，如*stock#，那么route key会被切分成单词来匹配他，*表示一个单词，#表示0或多个单词。
  • queue 队列 队列是RabbitMQ内部对象，存储消息。 > 队列和交换机有一些属性可以设置，设置完成后具有额外的属性。
  • 消息传递 消息有部分属性，可以设置，增加额外的功能。
