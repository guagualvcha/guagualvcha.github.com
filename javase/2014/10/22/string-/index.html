我对String的一点理解

String 是不可变类

  • string这个类非常特殊，String string="abc"，那么这个字符串内存是分配在常量池内的，但是如果 String string=new String("abc"),这个字符串对象是在堆上分配的。
  • String类实现了java中唯二的操作符重载 + 与 +=
  • string类提供的任何改变这个string对象的方法都会生成一个新的对象，如果没有任何改变仍然返回原来的对象以节省空间。
  • string类中改变字符串的操作都会new 出一个StringBuilder对象去构造，因此如果循环得改变这个字符串的话，最好代码里显式地去构造

一些陷阱

  • 在重写类的 toString 对象时，不要显式或隐式的调用this.toString()方法，会造成递归。
  • 多线程中使用线程安全的类StringBuffer

格式化输出

  • System.out.format("%d %s",1,"avc");
  • Formatter f=new Formatter(System.out);f.format("%d %s",1,"avc");
  • 格式化说明符： %[flag][width][.precision]conversion，flag默认为右对齐，加 - 可以左对齐，width指定最小长度，precision功能不太一致，在string中表示最大长度，在fload中表示小数点位数，在整数中不能使用，convert常用的如整数型d，浮点型f，字符串型s，科学计数型e。
  • String.format("%d %s",1,"avc")与Formatter用法一样，返回String类型，更加方便。

正则表达式的基础备忘

string对象支持的matches(),split(),repalce()都支持正则表达式。
